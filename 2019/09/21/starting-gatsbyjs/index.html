来康康Gatsby.js

发表于 2019-09-21 更新于 2019-09-23 分类于 闲扯

昨天发现notion用学生邮箱可以免费领personal plan，所以非常开心的玩了一会。东西确实挺不错的。

  • 完善的附件管理，包括图片，pdf，excel等
  • 并不太喜欢的block编辑方式，还是更喜欢markdown
  • 不错的风格
  • 强大的database
  • 不错的内容链接
  • 各种embed
  • 垃圾数学公式编辑，为啥编辑框不能换行

既然排版效果很好……这玩意，能不能做成个人博客。

去github上搜了一发，居然真的有，名字叫gine-blog。

已有项目初探

研究了这个小巧的项目之后，有些激动。它使用了目前很新的几个技术（相比Hexo等传统生成器）。

  • webpack
  • react
  • graphql

实际上，这些主要都来自另一个项目Gatsby.js，意在使用新技术快速构建网站。它能够汇总数据，使用graphql为react提供内容，并将内容固化，产生能够离开后端运行的纯网页。

非常适合作为个人博客生成使用。

之后我便忘了最开始的目的。

好像少了些什么

生成的方式似乎没有什么问题了，不过好像少了些什么。

gine-blog项目从notion拉取脏数据经过复杂处理填充到react中……作者实现了一个爬虫去从notion的网页上爬取源码…

我部署试了一下这个blog，效果还可以，也有些bug。比如bookmark丢失url，作者的文档新旧有些分不清楚。还是继续坚持hexo。

不过，这个gatsby.js有一些意思，可以研究一下。

关于blog

仔细思考了一下，相比于选择什么主题，选择什么模板，选择什么技术，这些都是次要的事情。最重要的是填充在里面的内容。所以这个事情就先这么放下了，如果要开始的话，有一系列的东西都需要学习，和我现在要干的事情有些冲突。

听说学长们在搞新的oj，给我的锅实际上很小，不需要负多大责任。那么oj也可以暂时放一放了。

blog继续维持使用hexo。

也许未来的哪天，会回来尝试一下使用rust+那时候的前端技术来构造一个不知道是什么的项目。
