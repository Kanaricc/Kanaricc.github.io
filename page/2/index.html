基于对Rust的浅薄理解，它一定是未来一门重要的语言。因此，为了进一步了解这哥们，对Rust的学习将逐步展开。

目前进度：慢速阅读第二项中


内存模型


梳理一下现存的内存管理模型。大致上分为两种：


    

  1. GC，例如Java与C#
    2. 

  2. 自行管理，例如c++
    3. 



这两种技术各有利弊。Rust提供了一种不同于以上两种办法的内存模型，所属权模型。一个内存有它的归属，对某个变量的读写需要涉及“借出”“还回”，又由此诞生“生命周期”的概念。通过这套系统，Rust实现了安全且无代价的内存管理办法。


所有权


    

  • Rust中变量默认不可变。
    • 



变量在rust中，还有着所有权的概念。


    

  • 当变量以传值方式传播，那么原有的所有权将会移动给新的变量名，原有变量名作废。
    • 

  • 变量可以创建只读引用，并用于传递。只读引用可以有多个。
    • 

  • 变量可以创建读写引用。只能有一个。
    • 

  • 只读与读写引用不可同时存在。
    • 



这样就从根源上解决了一些问题，例如多线程读写可能出现的各种问题。不过还是不是很清楚这种情况下要如何实现原本的功能。


生命周期


想象这样一种情况，“创建一个链表的迭代器”。一个必要的问题是确保使用迭代器时其所指向的链表没有出现问题（例如被销毁）。


在function的层级中，rustc能够自动推导出变量的传递关系与其最小生命周期。而程序上升到结构、架构、API上后，继续推导生命周期就不是很好做到，可能会耗费大量编译时间与资源。此时，让使用者主动标记生命周期会更有效率。尽管此时可能无法达到最小的目标。

Petya asks you to calculate the maximum possible score in the game that can be achieved by placing exactly one square.

You are given an array a1,a2,...,an(∀i∈[1,n],1≤ai≤n). Initially, each element of the array is unique.


Moreover, there are m instructions.


Each instruction is in one of the following two formats:


    

  1. (1,pos),indicating to change the value of apos to apos+10,000,000;
    2. 

  2. (2,r,k),indicating to ask the minimum value which is not equal to any ai ( 1≤i≤r ) and not less  than k.
    3. 



Please print all results of the instructions in format 2.


分析


这题强制在线.首先1操作相当于删除了这个数.


dalao自闭了一会get到了它的正确做法,我就直接拿来用了.


维护一权值线段树,位置i存其在a中出现的位置.那么当1到r区间内出现位置的最大值超过了r,根据鸽巢原理,至少有一个数未被限制.


加上不小于k的条件,就是k到r中,找到最小的一个r,使得它满足上面的条件,输出这个r.
